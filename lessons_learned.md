# Lessons Learned

Throughout this project, I reviewed my meeting notes from each iteration and reflected on the backend development process. This included API design, database integration, deployment, and testing. The experience helped me gain practical knowledge and identify both my strengths and areas that need improvement.

# 1. Importance of Proper Backend Planning

One of the most important lessons I learned is the importance of setting up a clear backend structure from the beginning. During Iteration 0, our team agreed on using a consistent endpoint (GET /products) and standardized field names.

This early alignment made the integration process much easier in later stages. It showed me that when multiple developers are working on the same system, having shared standards is essential for success.

# 2. Deployment and Cloud Integration Experience

This project allowed me to gain hands-on experience with deploying a backend application and integrating it with an online database. As mentioned in the meeting notes, we used MongoDB Atlas for database management and Render for deployment.

I also learned how to connect a GitHub repository directly to Render, which enabled automatic deployment whenever changes were pushed. This helped me understand how real-world backend systems are maintained and updated.

# 3. Importance of Testing and Integration

Another important lesson was understanding the role of testing in backend development. During Iteration 2 and 3, we focused on integrating multiple APIs and testing them.

We used Jest and Supertest to test endpoints and created an automate.js file to run all tests sequentially. This process showed me how important testing is to ensure that all components work correctly together.

# Areas for Improvement

# 1. Improving Knowledge of Testing Tools

During Iteration 2, I realized that writing and debugging tests took more time than expected. This showed that I need to improve my understanding of tools such as Jest and Supertest to become more efficient.

# 2. Better Time Management During Integration

Based on the meeting notes, the integration and testing phases took longer than planned. This showed me that I underestimated the complexity of combining multiple backend systems. In future projects, I will allocate more time for these stages.

# 3. Stronger Risk and Team Dependency Management

During the project, one team member was unable to continue, which created additional responsibilities for the remaining members. This situation showed me the importance of managing team dependencies and preparing for unexpected challenges.

# Conclusion

Overall, this project provided a realistic backend development experience. I learned how to design and build APIs, integrate them with a cloud database, deploy applications, and test them effectively.

By reflecting on my meeting notes, I gained a better understanding of both the technical and teamwork aspects of backend development. This experience will help me build more reliable and scalable systems in the future.
